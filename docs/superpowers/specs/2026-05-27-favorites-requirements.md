# 收藏功能 — 需求记录

> 日期：2026-05-27 | 版本：v1.0 | 类型：需求记录（非规格说明书）
> 关联：[AI 购房助手](2026-05-26-ai-assistant-requirements.md) — 收藏的 AI 对话控制

---

## 1. 概述

在平台中新增收藏功能，购房客户端用户可手动收藏感兴趣的楼盘，并在工作台查看收藏列表。

本模块为独立功能，不依赖 AI 助手。AI 助手可通过 Function Calling 间接操作收藏，但那是 AI 助手模块的职责，不在本文档范围内。

---

## 2. 功能清单

### 2.1 收藏按钮

| 维度 | 选择 | 说明 |
|------|------|------|
| 位置 | 房产详情页 | 详情页顶部或关键信息区域添加收藏按钮（心形图标） |
| 交互 | 点击切换收藏/取消收藏 | 实心/空心状态切换，即时反馈 |
| 未登录 | 不显示按钮 | 仅登录用户可见 |

### 2.2 收藏数据

| 维度 | 选择 | 说明 |
|------|------|------|
| 存储 | MySQL 新表（favorites） | 当前数据库无此表，需新建 |
| 字段 | customer_id, property_id, created_at | 基本结构，具体 DDL 在规格说明书中定义 |
| 唯一约束 | customer_id + property_id | 同一用户不能重复收藏同一楼盘 |

### 2.3 收藏列表展示

| 维度 | 选择 | 说明 |
|------|------|------|
| 位置 | Dashboard 底部 | 工作台底部展示收藏的楼盘列表 |
| 内容 | 名称 + 价格 + 区域 + 户型 + 缩略图 | 卡片样式 |
| 排序 | 按收藏时间倒序 | 最新收藏的排在前面 |
| 交互 | 点击跳转到房产详情页 | 与搜索结果卡片行为一致 |
| 空状态 | 显示友好提示 | 引导用户去浏览楼盘 |

### 2.4 后端 API

| 端点 | 方法 | 说明 |
|------|------|------|
| /api/favorites | GET | 获取当前用户的收藏列表 |
| /api/favorites | POST | 添加收藏（body: { propertyId }） |
| /api/favorites/{propertyId} | DELETE | 取消收藏 |

---

## 3. 与 AI 助手关系

本文档定义的收藏数据表和 API 同时服务于两个消费者：

- **前端页面** — 详情页按钮、Dashboard 列表
- **AI 助手** — 通过 Function Calling 调用同一套 API

双方共享同一份收藏数据，操作结果一致。
