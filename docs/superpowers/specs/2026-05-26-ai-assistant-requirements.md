# AI 购房助手 — 需求记录

> 日期：2026-05-26 | 版本：v1.2 | 类型：需求记录（非规格说明书）

---

## 1. 核心架构

### 1.1 LLM 与 AI 框架

| 维度 | 选择 | 说明 |
|------|------|------|
| LLM API | DeepSeek API | 对话（chat）+ 文本向量化（embedding） |
| Java AI 框架 | Spring AI | 通过 OpenAI Starter 适配，DeepSeek 兼容 OpenAI 协议，仅需修改 base-url |
| API Key 管理 | 环境变量 | 不硬编码，通过 `DEEPSEEK_API_KEY` 环境变量注入 |

### 1.2 RAG 知识检索

| 维度 | 选择 | 说明 |
|------|------|------|
| 检索引擎 | Apache Lucene 混合检索 | BM25（关键词）+ 向量（语义）混合评分 |
| 选择原因 | Chroma 无法嵌入 Java 进程，Lucene 纯 Java 原生运行 |
| Embedding 模型 | DeepSeek Embedding API | 与对话用同一家 API，统一管理 |
| Embedding 维度 | 1024 | DeepSeek embedding 模型输出维度 |

### 1.3 文档处理

| 维度 | 选择 | 说明 |
|------|------|------|
| 文档解析 | Apache Tika | Java 库，无需额外 Python 服务 |
| 支持格式 | PDF / Word | 开发商上传的常见文档格式 |
| 文档切分 | 固定 300 字一段 | 因为测试文档量不大（几百字），500 字太长，300 字合适 |
| 文件存储 | 本地两个文件夹 | `data/rag-platform/`（平台数据）+ `data/rag-uploads/`（上传文档），不存数据库 |

### 1.4 索引策略

| 维度 | 选择 | 说明 |
|------|------|------|
| 平台数据 | 启动时全量索引 | 从 `data/rag-platform/` 文件夹读取，应用启动时全量构建索引 |
| 上传文档 | 运行时增量索引 | 开发商上传的文档存入 `data/rag-uploads/`，上传后实时索引 |

两个文件夹用途：
- `data/rag-platform/` — 平台业务数据 + 房地产领域知识（静态，启动时读取）
- `data/rag-uploads/` — 开发商运行时上传的文档（动态，上传后增量索引）

### 1.5 前后端通信

| 维度 | 选择 | 说明 |
|------|------|------|
| 通信方式 | HTTP POST | 前端发消息 → 后端处理 → 返回 JSON 完整响应 |
| 输出方式 | 一次性返回 | 不做流式输出（打字机效果），实现简单 |
| 聊天历史 | 仅带最近 3 轮 | 不做 token 窗口自动截断，固定带最近 3 轮对话 |

## 2. 功能清单

### 2.1 界面

| 维度 | 选择 | 说明 |
|------|------|------|
| 入口形式 | 右下角悬浮气泡 | 类似常见在线客服样式 |
| 对话窗口 | 点击气泡展开小窗 | 气泡和窗口均固定定位、置顶（最高 z-index），不随页面滚动而隐藏 |
| 显示范围 | 登录后所有页面 | 客户端所有页面均可见；登录页不显示 |
| 输入方式 | 纯文本 | 不支持语音输入 |

### 2.2 对话管理

| 维度 | 选择 | 说明 |
|------|------|------|
| 会话模式 | 多会话持久化 | 类似 ChatGPT 的会话列表 |
| 默认行为 | 打开为空白新对话 | 顶部有按钮可展开历史会话列表 |
| 会话操作 | 创建 / 切换 / 删除 / 搜索 | 搜索功能用于在历史会话列表中查找 |
| 存储方式 | MySQL 新表 | 新建 session 和 message 两张表，不使用前端 localStorage |

### 2.3 AI 人格

| 维度 | 选择 | 说明 |
|------|------|------|
| 风格 | 亲切友好的购房助手 | 口语化、温暖、像朋友参谋购房，非正式房产顾问口吻 |

### 2.4 核心能力

| 能力 | 说明 |
|------|------|
| 问答 | 基于 RAG 检索结果回答用户问题 |
| 推荐房产 | 从对话中理解用户偏好，结合 RAG 检索和用户画像，推荐匹配楼盘 |
| 收藏房产 | 通过 Function Calling 执行收藏操作（用户说"帮我收藏"即可触发） |
| 查看收藏 | 通过 Function Calling 查询用户收藏列表，支持自然语言条件筛选 |
| 创建订单 | 通过 Function Calling 创建购房订单，AI 先展示订单摘要，用户确认后再执行 |

> AI 共五项能力：问答 + 推荐 + 收藏 + 查看收藏 + 创建订单。数据展示均以卡片形式在对话中输出。

### 2.5 查看收藏

| 维度 | 选择 | 说明 |
|------|------|------|
| 默认行为 | 返回最新 5 条收藏 | 按时间倒序 |
| 条件筛选 | 支持自然语言条件 | 如"看看我朝阳区的收藏""300万以内的收藏" |
| 数量上限 | 最多 5 条 | 即使符合条件的数据超过 5 条，也仅返回最新 5 条 |
| 展示形式 | 收藏卡片列表 | 嵌入 AI 回复消息中，卡片样式 |
| 收藏卡片内容 | 名称 + 价格 + 区域 + 户型 + 缩略图 | 与推荐卡片样式一致 |
| 卡片交互 | 点击跳转到房产详情页 | 与推荐卡片行为一致 |

### 2.6 创建订单

| 维度 | 选择 | 说明 |
|------|------|------|
| 流程 | AI 引导确认 | AI 收集信息 → 展示订单摘要 → 用户确认 → 执行创建 |
| 确认机制 | 用户说"确认"或类似表述 | 明确确认后才调用 Function Calling 创建订单 |
| 订单数据 | 复用现有下单逻辑 | 楼盘中已有的信息（价格、户型等），AI 从对话中提取填充 |
| 展示形式 | 订单摘要卡片 | 确认前以卡片展示即将创建的订单概要，供用户核对 |

### 2.7 推荐卡片与数据卡片

所有 AI 返回的结构化数据（推荐列表、收藏列表、订单摘要）统一使用卡片样式在对话中呈现，不输出纯文本列表。

**推荐卡片：**

| 维度 | 选择 | 说明 |
|------|------|------|
| 推荐逻辑 | 对话理解 + 用户画像，RAG 驱动 | 综合对话中提取的偏好 + 用户已有的偏好设置/浏览历史/收藏记录 |
| 展示位置 | 对话窗口内 | 嵌入在 AI 回复消息中 |
| 卡片内容 | 名称 + 价格 + 区域 + 户型 + 缩略图 | 卡片样式，非纯文本 |
| 卡片交互 | 点击跳转到房产详情页 | 卡片上无额外按钮 |
| 收藏触发 | 对话中自然语言 | 用户说"帮我收藏"即可触发 Function Calling |

**收藏卡片：** 与推荐卡片样式一致，同样点击跳转详情页。

**订单摘要卡片：** 展示楼盘名称、价格、户型、总价等关键信息，确认前以卡片呈现。

### 2.8 收藏功能

收藏的详情页按钮、数据存储、Dashboard 展示等独立功能见：[收藏功能需求文档](2026-05-27-favorites-requirements.md)

AI 助手通过 Function Calling 调用同一套收藏 API，AI 侧仅负责自然语言触发和结果以卡片呈现。

### 2.9 知识库

| 维度 | 选择 | 说明 |
|------|------|------|
| 组成 | 1) 平台业务数据 + 2) 房地产领域知识 + 3) 开发商自定义文档 | 三层知识来源 |
| 房地产知识获取 | 我帮你从公开渠道搜集 | 几百字左右的测试用文档，别太多别太少 |
| 开发商文档 | PDF / Word 上传，Apache Tika 解析 | 上传后自动切分、向量化、写入 Lucene 索引 |
| 权限 | 所有开发商共享 | 不做隔离，暂不考虑隐私问题 |

### 2.10 开发商知识库管理

| 维度 | 选择 | 说明 |
|------|------|------|
| 界面 | 开发商后台新增「知识库管理」页面 | 非 Profile 页中简单入口 |
| 功能 | 上传文档 / 删除文档 / 预览解析结果 | 完整 CRUD 子集 |
| 索引触发 | 上传即索引 | 实时同步处理，不做异步队列 |

### 2.11 异常处理

| 场景 | 处理策略 |
|------|------|
| DeepSeek API 不可用 | 告知用户"AI 助手暂时不可用，请稍后重试"，不做重试 |
| 问题超出知识范围 | 友好告知无法回答，不做兜底推荐 |

### 2.12 与现有功能关系

| 功能 | 关系 |
|------|------|
| 购房建议向导 (Wizard) | 独立共存，互不影响，不做任何改动 |

## 4. 技术决策记录

| 决策 | 选定 | 淘汰 | 原因 |
|------|------|------|------|
| 向量数据库 | Apache Lucene | Chroma | Chroma 是 Python 原生项目，无法嵌入 Java 进程 |
| Embedding 模型 | DeepSeek API | BGE 本地部署 | 统一管理，避免引入额外模型 |
| 文档切分 | 固定 300 字 | 按段落 / 混合切分 | 测试文档量小，简单方案足够 |
| 前后端通信 | HTTP POST | SSE / WebSocket | 不做流式，JSON 一次性响应最简 |
| 输出方式 | 一次性返回 | 流式逐字 | 实现简单 |
| 历史消息 | 固定 N 轮 | Token 窗口截断 | 实现简单 |
