## Epic 1: Backend Infrastructure (Sprint 1)

### Rationale

Before any frontend work, we need the data layer and APIs to support:
- Price history storage and querying
- Personalized recommendation logic
- These are foundational for all frontend features

### Task Breakdown

#### 1.1 New Database Table: `price_history`

**File**: `backend/src/main/resources/db/schema.sql`

```sql
CREATE TABLE IF NOT EXISTS price_history (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    property_id BIGINT NOT NULL COMMENT '房产ID',
    record_date DATE NOT NULL COMMENT '记录日期（月份粒度）',
    price_per_sqm DECIMAL(12,2) NOT NULL COMMENT '当月单价',
    total_price DECIMAL(14,2) NOT NULL COMMENT '当月总价',
    source VARCHAR(50) DEFAULT 'system' COMMENT '数据来源',
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_property_date (property_id, record_date)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='房价历史记录';
```

#### 1.2 New Entity: `PriceHistory`

**File**: `backend/src/main/java/com/fangyicha/entity/PriceHistory.java`

Standard entity with: id, propertyId, recordDate (LocalDate), pricePerSqm (BigDecimal), totalPrice (BigDecimal), source, createdTime.

#### 1.3 New Mapper: `PriceHistoryMapper`

**File**: `backend/src/main/java/com/fangyicha/mapper/PriceHistoryMapper.java`

- Extends `BaseMapper<PriceHistory>`
- Custom query: `selectByPropertyIdAndMonths(@Param("propertyId") Long propertyId, @Param("months") Integer months)` using `DATEDIFF` or `DATE_SUB`

#### 1.4 New Service: `PriceHistoryService`

**File**: `backend/src/main/java/com/fangyicha/service/PriceHistoryService.java`

- Interface: `List<PriceHistory> getHistoryByProperty(Long propertyId, Integer months)`
- Implementation: `PriceHistoryServiceImpl` - queries mapper, orders by record_date ASC

#### 1.5 New Controller Method: `GET /api/properties/{propertyId}/price-history`

**File**: `backend/src/main/java/com/fangyicha/controller/PropertyController.java`

- Add endpoint to existing PropertyController
- `@PreAuthorize("hasRole('CUSTOMER')")` or permitAll (public)
- Returns `List<PriceHistory>` sorted by date ascending

#### 1.6 New Service: `CustomerRecommendationService`

**File**: `backend/src/main/java/com/fangyicha/service/RecommendationService.java`

Interface:
```java
List<RecommendationDTO> getRecommendations(Long customerId, int limit);
```

Implementation logic:
1. Load customer profile (preferredLocations, budgetMin, budgetMax)
2. If preferredLocations is blank, return random 6 '在售' properties
3. Parse preferredLocations by comma, build LIKE query for each
4. Query properties matching ANY preferred location + status='在售'
5. Calculate budget midpoint: (budgetMin + budgetMax) / 2
6. Sort by ABS(totalPrice - budgetMidpoint)
7. Limit to `limit` (default 6)
8. For each result, determine matchReason:
   - if matched by location: "在您的偏好区域"
   - if within budget range: "符合您的预算范围"
   - if both: "高度匹配您的需求"
   - default (random): "热门推荐"

#### 1.7 New DTO: `RecommendationDTO`

**File**: `backend/src/main/java/com/fangyicha/dto/RecommendationDTO.java`

Fields: id, propertyName, location, floorPlanType, areaSqm, totalPrice, pricePerSqm, status, imageUrls, matchReason, developerId, developerName.

#### 1.8 New Controller Method: `GET /api/customers/recommendations`

**File**: `backend/src/main/java/com/fangyicha/controller/CustomerController.java`

- New endpoint in existing CustomerController
- `@PreAuthorize("hasRole('CUSTOMER')")`
- Query param: limit (default 6)

#### 1.9 Seed Data Generation Script

**File**: `backend/src/main/resources/db/data.sql`

- For each of ~120 properties, generate 24 monthly price records
- Start from 2024-06, end at 2026-05
- Base prices from `Property.pricePerSqm` and `Property.totalPrice`
- Apply random variation: -10% to +5% over 24 months (simulate gradual increase)
- Use MySQL `INSERT INTO price_history` statements

#### 1.10 Add Developer Name to Property Detail API

**File**: `backend/src/main/java/com/fangyicha/controller/PropertyController.java` or a new DTO

Modify `GET /api/properties/{id}` to also return `developerName` by joining with developer table. Use a wrapper DTO `PropertyDetailDTO` or add field to existing Property entity (via left join in a custom mapper method).

### Dependencies

- None (works on current 1.4 branch state)

### Risks

- Seed data: generating 120 * 24 = 2880 rows could make the SQL script large. Consider a programmatic seed in Java (CommandLineRunner) instead.
- Recommendation query performance: using LIKE for location matching may be slow with large datasets. Consider adding MySQL FULLTEXT index or using FIND_IN_SET.

## Epic 2: Property Detail Page Enhancement (Sprint 2)

### Rationale

The property detail page is the most-viewed page. This sprint delivers the biggest visual impact: image carousel, price trend chart, and map.

### Task Breakdown

#### 2.1 New Component: `PropertyCarousel.vue`

**File**: `frontend/src/components/property/PropertyCarousel.vue`

Props:
- `images: string[]` -- array of image URLs

Behavior:
- If images empty, show placeholder with `el-empty` (not gradient)
- Use `el-carousel` with `height="400px"`, `indicator-position="none"`
- Custom navigation arrows (left/right buttons)
- Custom bottom thumbnail strip: small squares, active one highlighted
- Hover pauses autoplay (default interval 4000ms)
- Keyboard support via global keydown listener (left/right arrow keys)
- Image object-fit: cover, rounded corners at top of card

States:
- Loading: skeleton block matching carousel dimensions
- Empty: placeholder image with house icon
- Error per slide: show broken image fallback

#### 2.2 New Component: `PriceTrendChart.vue`

**File**: `frontend/src/components/property/PriceTrendChart.vue`

Props:
- `data: PriceHistoryItem[]`

Behavior:
- Use `vue-echarts` (VChart) with ECharts 6.1.0
- Dual Y-axis: left for pricePerSqm, right for totalPrice
- X-axis: month labels (format: "YYYY-MM")
- Two line series with smooth curves
- Tooltip on hover showing both values
- Area fill below lines (semi-transparent)
- DataZoom slider at bottom for time range selection
- Responsive: resizes with window (ResizeObserver)
- Chart animation on initial render

States:
- Loading: skeleton block matching chart dimensions
- Empty: "暂无价格数据" with chart icon

Helper:
- Parse `recordDate` using dayjs for formatting
- `pricePerSqm` displayed to 2 decimal places with ¥ prefix

#### 2.3 New Component: `PropertyLocation.vue`

**File**: `frontend/src/components/property/PropertyLocation.vue`

Props:
- `longitude: number`
- `latitude: number`
- `location: string` (text address)

Behavior:
- Display options (choose based on availability):
  1. **Primary**: Embed static map image URL (from Gaode static map API or similar) showing a marker at the coordinates
  2. **Fallback**: If no API key, show a styled div with the address text + a "查看地图" button that opens `https://uri.amap.com/marker?position={lng},{lat}` in new tab
  3. **Empty**: If no coords, show address text only with "暂无位置信息"
- Show address text below/overlaid on the map area
- Mark on the map using a red pin marker overlay

States:
- Loading: skeleton square matching container
- Empty (no coords): Text display with location string
- No API key: Graceful fallback to text + external link

## Epic 3: Dashboard Recommendation + Location Editor (Sprint 3)

- `update:modelValue`

Behavior:

3. Current `1.4` branch is the base; each sprint should be committed with clear messages for easy revert
